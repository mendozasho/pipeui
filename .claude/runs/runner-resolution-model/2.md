# Slice 2 — Join honors use_transformed
Issue: #12 (sub-issues #16 business_logic, #17 api, #18 ui)   Branch: feature/runner-resolution-model-2

## Plan (written before coding)

### Approach per layer

**business_logic (#16, acceptance 0,1,2,5)** — `src/pipeui/workflow/builtins.py`
- `_execute_join` currently hardcodes `instance_table_name(right_source_id)` (raw). Route the
  right-source read through `resolve_frame(conn, right_source_id, mode)` where
  `mode = TRANSFORMED if cfg.get("use_transformed") else RAW`.
- `resolve_frame` returns `(frame, ref)`. For RAW the frame is the instance table contents; for
  TRANSFORMED it's the latest/materialized staging frame (run-once-if-absent, cycle-guarded).
  Register the returned frame as a temp view (same pattern as the existing left_df view) and join
  the left view against the right view — so both raw and transformed paths use a uniform
  "frame -> temp view -> SQL join" shape. This keeps `_execute_join` agnostic to *where* the right
  frame came from, which is the whole point of the seam.
- `_validate_join_config` only validates config shape today (no table read); it stays shape-only.
  The acceptance for it is the resolve routing, which lives in `_execute_join`. No behavior change
  needed in the validator beyond confirming it does not block use_transformed configs.
- use_transformed=false path must be byte-for-byte the prior behavior (acceptance 0).

**api (#17, acceptance 3)** — right-column-fetch endpoint
- The join modal fetches right-source columns. Add an endpoint that returns the *transformed*
  column set when a transformed flag is set, raw columns otherwise. Put it on the sources router
  as `GET /sources/{source_id}/join-columns?transformed=true|false` (sources.py owns source-column
  reads; raw path reuses the existing column_registry query, transformed path reads the resolved
  transformed frame's columns via resolve_frame).
- raw -> column_registry columns (column_id/column_name/column_type), matching `/sources/{id}`.columns.
- transformed -> columns of the resolved transformed frame (names only; type best-effort from dtype).

**ui (#18, acceptance 4)** — `src/pipeui/frontend/screen-builder.jsx` (regression + wiring)
- JoinModal already sends `use_transformed` and calls `fetchRightColumns(rid, ut)`. The wiring fn
  `fetchJoinRightColumns(rightSourceId)` ignores the 2nd arg and always hits raw `/sources/{id}`.
  Fix it to accept `(rightSourceId, useTransformed)` and call the new join-columns endpoint with the
  flag. Frontend tests assert the regression: modal sends use_transformed=true and requests the
  transformed column set when the toggle is on.
