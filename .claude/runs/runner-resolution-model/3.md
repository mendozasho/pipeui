# Slice 3 — Uniform StepExecutor contract + step-type registry
Issue: #13 (sub-issues #19, #20)   Branch: feature/runner-resolution-model-3

## Plan (written before coding)
- Approach per layer (business_logic only):
  - **context.py (#19):** `StepContext` dataclass carrying one step's execution
    properties — the exact keys the step dict held — plus a `step_type`
    discriminator. Factory classmethods build it over the existing map-table dicts
    produced by the runner's fetchers:
      - `from_function(step_dict)` / `from_set(step_dict)` — over a
        `_fetch_steps` row (`source_function_map`); both wrap the same function
        step shape (a set is the function-step container), differing only in the
        `step_type` tag they stamp (`"function"` vs `"set"` are aliases of the
        same execution path here; `from_set` is the set-origin name).
      - `from_builtin(step_dict)` — over a `get_builtin_steps` row
        (`source_builtin_map`).
    The context exposes `.step_type`, `.position`, and the wrapped `.data` dict so
    executors read the identical fields the inline loop reads today — behavior-
    preserving by construction.
  - **executors.py (#20):** a `StepExecutor` contract (`execute(...)` returning a
    uniform `StepExecResult` carrying the new working frame, the list of step-
    result dicts, and whether staging was written) with `FunctionStepExecutor`
    and `BuiltinStepExecutor`, registered in a `STEP_EXECUTORS` dict keyed by
    step_type. Each executor wraps the EXISTING helpers (`_execute_transform_step`,
    `_execute_validation_step`, `execute_builtin_step`) and emits the SAME entries
    the inline loop emits today.
  - **run.py (#20):** replace the inline `if step_type == "builtin" ... continue`
    + function-step blocks in `run_pipeline`'s loop with: build a `StepContext`
    per step, look up its executor in the registry, call it, splice its results.
    No `if/elif` on step type remains in the loop.
- Utilities/patterns reused:
  - `pipeui.workflow.run._execute_transform_step / _execute_validation_step /
    _transform_runresult / _builtin_result / _write_staging_table / _step_has`
  - `pipeui.workflow.builtins.execute_builtin_step / get_builtin_steps`
  - test seeds in `tests/test_run_workflow.py`: `_seed_transform_step`,
    `_seed_validation_step`, `_seed_mixed_set`, `attach_builtin`, `_val_col`,
    `_register_multicol_source_and_ingest`, `_list_staging_tables`.
- Test command: `PYTHONPATH=src .venv/bin/pytest tests/test_run_workflow.py -q`
  (green gate: FULL suite `PYTHONPATH=src .venv/bin/pytest -q` — baseline
  473 passed + 1 known pre-existing macOS OOM failure in test_worker.py).
