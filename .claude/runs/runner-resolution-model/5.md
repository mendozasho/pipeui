## Plan (written before coding)

### Honest framing (verified against the checkout before planning)
- **Migration (#21):** Slice 3 already replaced the inline `if/elif` step dispatch in
  `run_pipeline` with the `STEP_EXECUTORS` registry (`workflow/executors.py`,
  `workflow/context.py`), and slice 4 added the function-set adapter. Every
  runner-dependent API endpoint already routes through the unified model via
  `run_pipeline` / `run_set_across_sources` / `run_validation_across_sources` /
  `get_staging_rows`:
    - `POST /pipelines/{id}/run` -> `run_pipeline`
    - `GET  /pipelines/{id}/staging` -> `get_staging_rows`
    - `GET  /pipelines/{id}/export/results` -> `run_pipeline` -> `build_results_report`
    - `GET  /pipelines/{id}/export/transformed` -> staging table (written by the registry executors)
    - `POST /pipelines/run-set` -> `run_set_across_sources` -> `run_pipeline`
    - `POST /validations/{fn}/run` (api/validations.py) -> `run_validation_across_sources` -> `run_pipeline`
  No endpoint bypasses the unified model. The acceptance is therefore the *passing
  behavior-preservation test* asserting each endpoint's response shape/content is
  unchanged AND that it flows through `STEP_EXECUTORS` (so a future regression that
  re-introduces a bypass path breaks the test). No rewrite is invented.

- **Removal (#22):** Proven by grep that EVERY top-level def in `workflow/run.py`
  retains at least one live caller after the registry swap (the executors call
  `_execute_transform_step`, `_execute_validation_step`, `_transform_runresult`,
  `_builtin_result`, `_write_staging_table`; those transitively use the rest).
  `execute_builtin_step` (builtins.py) is STILL LIVE (called by `BuiltinStepExecutor`).
  There is **no execution code left dead by this refactor to remove**. This is the
  explicitly-blessed "little is genuinely dead" outcome: removing nothing, covering
  the no-bypass behavior with a test, and recording the grep evidence. No live code
  is deleted to hit a quota.
