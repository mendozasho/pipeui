# Slice 1 — numeric-column-binding
Issue: #99 (sub-issues #101 backend, #102 UI)   Branch: feature/param-binding-output-mode-1

## Plan (written before coding)
- **Approach per layer:**
  - **business_logic (#101, climb data→business):**
    - `classification.py`: add ONE pure leaf helper `binding_kind(type_str)` derived
      off the EXISTING `function_class` — `scalar` → `"value_or_column"`,
      `pd.Series` → `"column_only"`, `pd.dataframe` → `"table"`. DB-free. Built off
      `derive_function_class([type_str])` so a single param type maps to its class,
      then class→kind. Unknown type raises (callers gate with `is_known_param_type`).
    - `suggest.py`: DELETE `_SUGGEST_TYPES` / `_SCALAR_TYPES`. Derive eligibility from
      `binding_kind`: a param is included unless its kind is `"table"` (pd.DataFrame).
      `param_kind` = `"column"` when kind is `column_only`, else `"scalar"` (numeric/str
      text). Emit a new `binding_kind` field per param. `value_or_column` numerics get
      `suggested_columns` + `current_bindings` (alias_map.position order) exactly like str.
    - `attach.py`: DELETE `_REQUIRES_BINDING`. Generalize `_find_missing_bindings`: a
      param needs a binding ONLY when `binding_kind == "column_only"` (pd.Series) OR it
      is `value_or_column` with NO literal AND NO has_default; satisfied by binding OR
      literal OR has_default. `_write_alias_rows`: write alias rows for any param that
      HAS a binding (not gated on a literal type set) — so column-bound numerics persist.
      Add `has_default` to the param_rows query so the rule can read it. The existing
      equal-length-among-varying guard (`_validate_bundles`/`pair_bundles`) already
      covers numeric varying columns — no change needed there.
  - **ui (#102):** `screen-builder.jsx` `PendingStepCard`: generalize the str-only
    conditionals to the backend-emitted `binding_kind` via a SINGLE predicate
    (`isValueOrColumn(p)` = `p.binding_kind === "value_or_column"`; `isColumnOnly` =
    `"column_only"`). Reuse the EXISTING `strModes`/`isColumnMode`/`handleSave`
    machinery, renamed conceptually to "toggle modes" but keep `strModes` state keyed by
    param_id for ALL value_or_column params (not just str). text→column clears the stale
    scalar (mirror existing str behavior). Save-guard: a value_or_column param with no
    value, no column, and no `has_default` blocks Save (mirror backend). replace-target
    picker already iterates `params.filter(isColumnMode)` — once a numeric is column-bound
    it appears automatically.
- **Utilities/patterns reused:**
  - `classification.derive_function_class` (class derivation), `is_known_param_type`.
  - attach: `pair_bundles`/`BundleLengthError` (equal-length guard), `_write_alias_rows`,
    `_find_missing_bindings`, `_has_literal` pattern.
  - suggest: existing unified param loop, `current_bindings` ORDER BY am.position.
  - frontend: `strModes`, `isColumnMode`, `handleSave`, `initScalarValues`,
    `initSelections`, the existing `["text","column"]` toggle buttons.
