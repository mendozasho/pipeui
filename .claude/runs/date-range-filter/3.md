# Slice 3 — Attach boundary + end-to-end run
Issue: #118 (sub-issues #123, #124)   Branch: feature/date-range-filter-3

## Plan (written before coding)
- Approach per layer (data / business / api / ui):
  - **business (domain/functions/builtins.py):** add a write-boundary validation seam to the
    builtin registry — a new optional `boundary_validate: Callable[[conn, source_id, cfg], str | None]`
    field on `BuiltinSpec` (default `None`), mirroring the existing OCP pattern (#50): a
    DB-aware check is a registration, not a type-specific branch. Register
    `_date_range_boundary_check` on the `date_range` spec: it re-runs the pure shape
    validator (the patch path never ran it), then checks every condition column against
    `column_registry` for the source — type must be in `{DATE, TIMESTAMP, TIMESTAMPTZ}` —
    returning a rejection message that names the offending column (unregistered columns
    are also rejected by name). `attach_builtin` calls `spec.boundary_validate` after the
    source-exists + singleton checks. `patch_builtin` gains the same call when
    `builtin_config` is provided; its return contract changes from `bool` to
    `dict | None` (`{"ok": True}` / `{"ok": False, "detail": ...}` / `None` = not found)
    so the rejection message can reach the API seam — the same shape `attach_builtin`
    already uses. Non-date_range types register no boundary check, so their attach/patch
    behavior is byte-identical at the HTTP seam.
  - **api (middleware/builtins.py):** patch route maps the new contract:
    `None` -> 404 (unchanged), `{"ok": False, ...}` -> 422 JSON body (the same rejection
    contract the attach route already emits — the "detail message shape" surface slice 4
    reads), `{"ok": True}` -> 200 (unchanged). Attach route needs no change (already
    returns 422 on `ok=False`). GET /sources/{sid}/pipeline needs no change (generic).
  - **data / ui:** none.
- Utilities/patterns reused (file + symbol):
  - `BUILTIN_EXECUTORS` / `BuiltinSpec` registry (builtins.py, #50) — boundary check as spec metadata.
  - `_validate_date_range_config` (slice 2) — re-used as the shape gate inside the boundary check.
  - Route-test pattern: FastAPI app + `dependency_overrides[get_conn]` (tests/test_builtins.py::test_get_builtins_endpoint, tests/test_api_pipelines.py::client).
  - End-to-end pattern: `_make_csv` + `create_source` + `ingest_source` + run + staging read-back (tests/test_api_staging.py::_register_and_ingest) — replicated in test_api_pipelines.py (the ledger's test surface for this slice).
  - `content_hash_id` source/column seeding helper pattern (tests/test_builtins.py::_make_source) — extended locally with per-column types.
- Test command (for red-green cycles):
  `PYTHONPATH=src <repo-venv-python> -m pytest tests/test_builtins.py tests/test_api_pipelines.py -q`
